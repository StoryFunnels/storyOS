The MCP shipped stdio-only (`node dist/index.js` with a single env token) — fine for
local Claude Desktop/Code, wrong for a cloud product: you can't hand a local file path
to a teammate. Add a hosted Streamable HTTP endpoint so anyone connects with a URL + PAT.

- `packages/mcp/src/http.ts` — stateless Streamable HTTP server (`POST /mcp`, `GET /health`).
  **Per-request auth**: each call carries its own `Authorization: Bearer mn_pat_…`, so one
  endpoint serves every user; the API scopes each response. No shared token.
- `client.ts` / `server.ts` refactored so `buildServer(client)` accepts a per-request client;
  stdio path (env token) unchanged.
- `docker/mcp.Dockerfile` + `mcp` service in `docker-compose.yml` (STORYOS_URL → `http://api:3001`).
- Caddy TLS drop-in routes a subdomain (`mcp.<domain> → mcp:3002`).
- README documents the hosted config; stale `os.jamescookmedia.com` refs fixed to `app.storyos.dev`.

Verified locally: health 200, missing-token → 401, and a real MCP client handshake over
HTTP lists 13 tools + reads live records.
