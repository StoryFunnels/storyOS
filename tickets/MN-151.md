## Problem
The "Cards" popover (board/gallery/list) lets you toggle which fields show, but not the
**order** they appear on the card. The order is fixed regardless of how the user wants to
read the card.

## What
- Make the FIELDS list in the Cards popover drag-reorderable (a drag handle per row).
- Card rendering follows `card_field_ids` order; reordering updates that array.
- Applies to board, gallery and list card bodies (and calendar chips).

## Acceptance
- [ ] Drag a field up/down in the Cards popover and the card layout reflects the new order.
- [ ] Order persists with the view (via card_field_ids).
