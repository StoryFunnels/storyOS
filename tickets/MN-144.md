## The leak

Integration credentials are stored in the workspace `settings` JSONB blob
(`settings.linear.api_key`, `settings.github.token`). `WorkspacesService.listForUser`
returned the row with `{ ...w }`, so the **raw keys were serialized into every
`GET /workspaces` response** — and therefore surfaced to the MCP (`list_workspaces` /
`get_started`) and to anyone who could read workspace settings. Discovered when the
hosted MCP returned a live Linear key in plaintext to an agent.

Note the integration endpoints themselves were already safe — `getConfig` returns
`{ has_key: boolean, team_keys }`, never the secret. Only the workspace serializer leaked.

## Fix

- `apps/api/src/common/redact-secrets.ts` — recursive redactor that masks secret-named
  keys (`api_key`, `token`, `secret`, `password`, `accessToken`, `clientSecret`, …) to
  `"[redacted]"` while preserving identifier siblings (`team_keys`, `repos`). Matches by
  normalized key name, so **new integrations are covered automatically**.
- Applied at every workspace serialization boundary in `WorkspacesService`
  (`listForUser`, `create`, `update`).
- Regression test in `redact-secrets.test.ts`.

## Operational

- **The exposed Linear key must be rotated** in Linear — it was shown in plaintext and
  is compromised regardless of the code fix. Re-save the new key via the integrations UI.

## Acceptance
- [x] `GET /workspaces` (and the MCP) never returns integration secrets.
- [x] Non-secret settings (team_keys, repos, theme) preserved.
- [x] Regression test covers snake_case, camelCase, empty, and identifier-like keys.
- [ ] Deployed (rebuild api) + Linear key rotated.

## Follow-ups (harden further — separate tickets)
- Encrypt integration secrets at rest (app-level AES-GCM with a key from env), so a DB
  dump or backup never exposes them — redaction only covers the response path.
- Consider moving integration creds out of `settings` into a dedicated, write-only
  `integration_credentials` table that no general serializer ever selects.
- Add a response-layer guard (interceptor) that runs `redactSecrets` on all payloads as
  defense-in-depth, not just the workspace serializer.
