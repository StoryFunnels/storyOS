## Problem
View edits (filters, sorts, grouping, card fields, size) are held as a local draft until you
press **Save to view** — easy to lose changes and confusing ("why is there a Save button?").

## What
Auto-persist view configuration changes so tweaks stick without a manual save.

Design decision to make (flag in the ticket, not assume):
- **Option A — full auto-save**: every change writes to the shared view immediately. Simplest;
  but a member's ad-hoc tweak changes the view for everyone.
- **Option B — auto-save personal, explicit "share"**: changes auto-save to a *per-user* view
  state; "Save to view" (or "Update shared view") only when publishing to the team. Preserves
  the original C11 intent (ad-hoc tweaks don't touch the shared view) while removing the
  friction for solo/owner use.

Recommend **B** for shared workspaces, **A**-like behavior for a solo owner. Debounce writes;
show a subtle "Saved" indicator instead of a button.

## Acceptance
- [ ] View changes persist automatically (no data-loss from forgetting to save).
- [ ] A clear, low-friction model for personal vs shared view state (per decision above).
- [ ] Subtle saved-state feedback replaces the prominent Save button.
