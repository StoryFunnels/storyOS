## Problem
After importing Issues from Linear, a workflow state appears lost — issues that were
"In Progress" in Linear didn't carry that state through (mismapped to a default or blank).

## Investigate / fix
- Verify the import maps Linear workflow states 1:1 onto the target select field's options
  (create missing options rather than dropping to a default).
- Confirm state **type/category** (backlog / unstarted / started / completed / canceled)
  and per-issue current state are both preserved.
- Re-import should update, not duplicate or reset, an issue's state.

## Acceptance
- [ ] Every Linear workflow state exists as an option and imported issues land in the
      correct one — no issue silently loses "In Progress" (or any) state.
- [ ] A re-import preserves states.
